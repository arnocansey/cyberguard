1. Introduction

CyberGuard is an AI-powered cybersecurity threat detection and incident response platform. It ingests logs, classifies threats, raises alerts, and supports full incident lifecycle management.

2. Access and Roles

  • Admin: admin@cyber.local / ChangeMe123!
  • Security Analyst: analyst@cyber.local / ChangeMe123!

ADMIN has full access; SECURITY_ANALYST has SOC operational access.

3. Quick Start

  1. Start backend, frontend, and AI service.
  2. Log in with role-appropriate credentials.
  3. Upload logs from the Logs page.
  4. Triage detections in Threats.
  5. Create and close incidents in Incidents.

7. Troubleshooting

  • Missing token: ensure Bearer token is attached.
  • Forbidden: ensure account has required role.
  • ENOTFOUND ai-service: verify AI service URL and runtime.

8. Daily Operations Checklist

  1. Review dashboard for spikes.
  2. Triage new alerts.
  3. Create incidents for high/critical threats.
  4. Use Copilot for response guidance.
  5. Close incidents with clear notes.
  6. Review audit trail before shift close.
